# Harness Effect Measurement (promptfoo)

역추적 규칙의 실제 효과를 측정하는 promptfoo 테스트.

## 측정 목적

"역추적 규칙을 추가했을 때 LLM 응답이 실제로 바뀌는가?"

| 변형 | 설명 |
|------|------|
| 하네스 OFF | 단순 "이 버그 고쳐줘" |
| 하네스 ON | 역추적 3단계(증상/경로/수정위치) 강제 후 수정 |

## 테스트 결과 (Gemini 2.5 Flash, 2026-03-16)

| 변형 | 통과 | 실패 | 통과율 |
|------|------|------|--------|
| 하네스 OFF | 8/10 | 2/10 | 80% |
| 하네스 ON | 9/10 | 1/10 | 90% |

**ON이 OFF보다 10% 앞섬.**

### OFF 실패 케이스
- TC01 (파일 확장자 하드코딩): 경로 추적 불충분 - ON이 잡음
- TC09 (API 키 하드코딩): "이미 노출된 키 즉시 폐기" 미언급

### ON 실패 케이스
- TC09 (API 키 하드코딩): 동일 - 역추적 규칙이 보안 대응 액션까지는 강제 안 함

### 해석
- 역추적 규칙은 "경로 추적"을 명시적으로 강제하여 TC01 같은 흐름 분석 케이스에서 차이를 만듦
- 유능한 모델 + 단순 버그에서는 차이가 작음 (OFF도 기본 근본원인은 식별)
- **에이전트 세션 / 복잡한 다단계 버그에서 차이가 더 클 것으로 예상**

## 테스트 케이스

실제 발생한 버그 10개 (patterns.json 기반):

| # | 케이스 | 근본원인 유형 |
|---|--------|-------------|
| TC01 | 파일 확장자 하드코딩 | 설계 결함 |
| TC02 | Windows cp949 vs UTF-8 충돌 | 환경 특화 |
| TC03 | 부분 Edit으로 기능 손실 | 프로세스 결함 |
| TC04 | try 블록 안 표준 라이브러리 import | 스코프 오해 |
| TC05 | import os 누락 | 단순 누락 |
| TC06 | JSON 배열 덮어쓰기 | 도구 사용 오류 |
| TC07 | TypeScript 타입 일부 누락 | 전파 누락 |
| TC08 | 배치 선택 항상 동일 50개 | 스키마 불일치 |
| TC09 | API 키 하드코딩 | 보안 위반 |
| TC10 | subprocess 버퍼링 | 환경 특화 |

## 실행 방법


## 평가 기준

각 케이스에 `llm-rubric` 평가자 적용 (Gemini 2.5 Flash):
- 근본원인 식별 여부 (증상 패치 vs 근본 수정)
- 경로 추적 여부: "입력 → 처리 → 실패" 흐름을 명시했는가 (복잡한 케이스 5개)

## 한계

- promptfoo는 프롬프트 수준 테스트 (프로덕션 효과 검증 아님)
- "역추적 규칙이 응답 품질을 바꾸는가"를 측정하는 도구
- 실제 효과는 제어 실험(실 세션 비교)에서 측정 필요
- TC09처럼 루브릭이 보안 액션 전체를 요구하면 ON/OFF 둘 다 실패 가능
